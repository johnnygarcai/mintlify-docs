A **project** is one impact initiative — a well-drilling program, a school build, a food distribution effort. Projects own everything downstream: reporting tasks, field reports, metrics, and donor reports.

## Create a project

From your organization's **Projects** page, create a new project and fill in:

- **Title** — e.g. "Clean Water Wells — Rural Rwanda".
- **Summary** — the elevator pitch. This appears on donor-facing pages, so write it for donors, not for internal use.
- **Hero image** — the visual anchor for the project and its donor reports.
- **Location** — an address or place; projects appear on maps in both the dashboard and the reporter app, and reporters can get directions to the site.
- **Metrics** — the numbers this project will prove. This is the most important design decision you'll make; see [Defining metrics](/projects/defining-metrics).
- **SDG goals** — tag the project with the UN Sustainable Development Goals it advances (1–17).
- **Funding** — an optional funding goal and amount raised, tracked manually, for display alongside impact data.

## Project status

| Status | Meaning |
| --- | --- |
| **Draft** | Being set up; not yet collecting reports. |
| **Active** | Live — reporters see its tasks and data is flowing. |
| **Paused** | Temporarily on hold. |
| **Archived** | Finished; kept for the record. |

## Guiding the AI (optional)

Project settings include an optional **custom AI guidance prompt**. Anything you write there is taken into account when Mirage's AI analyzes this project's reports and writes its donor narratives — useful for domain-specific framing ("water access should be evaluated against WHO standards", "always refer to participants as 'students'"). Organizations working with the Gloo Human Flourishing framework can also enable its values rubric per project.

## After creation

The project page becomes your hub for that initiative:

- **Reporting tasks** — the work you're asking reporters to do. See [Reporting tasks](/projects/reporting-tasks).
- **Reports** — submissions from the field, with review status.
- **Donor reports** — every impact page generated for this project.
- **Revisits** — scheduled follow-up verifications. See [Revisit tasks](/projects/revisit-tasks).

<Tip>
  Keep projects focused. One project = one story you want to tell donors. If two efforts would need different metrics or different donor narratives, make them two projects.
</Tip>
